# Optimization Roadmap

## AuthServer + Provider Foundation

- [x] Add a focused implementation slice plan for AuthServer client consolidation.
- [x] Introduce shared typed AuthServer response DTO parsing and normalization helpers.
- [x] Add a centralized `AuthServerClient` in main process with timeout + retry behavior.
- [x] Refactor mail OAuth flow to use the shared AuthServer client for start/exchange/refresh orchestration.
- [x] Add provider contract interfaces + `ProviderManager` foundation for mail drivers.
- [x] Move account sync resolution behind provider manager contracts.
- [x] Add provider-manager account-level sync module resolution for `emails`/`contacts`/`calendar`/`files`.
- [x] Route DAV execution through provider-manager module checks instead of unconditional sync attempts.
- [x] Convert provider manager to a registry-based driver resolver with graceful resolution errors.
- [x] Expose account provider capability lookup through main IPC → preload → renderer client.
- [x] Add provider-managed email sync service resolution and remove inline worker orchestration from account IPC.
- [x] Clear architecture check blocker in add-account provider card styling (`bg-white/20`).
- [x] Add provider-managed ancillary sync service resolution (contacts/calendar) and remove direct `syncDav` orchestration from account IPC loop.
- [x] Add module-level sync outcomes (`emails`/`contacts`/`calendar`/`files`) to account sync summaries for partial-success visibility.
- [x] Add explicit partial-success fields (`partialSuccess`, `failedModules`) to account sync summaries.
- [x] Add normalized provider sync error shape with category/retryability and include it in sync-status error broadcasts.
- [x] Update renderer sync listeners to consume structured `syncError` payloads (message/category) before raw error text.
- [x] Surface partial-success sync outcomes in mail/contacts status text using `summary.partialSuccess` and `failedModules`.
- [x] Surface partial-success sync outcomes as system toasts with failed-module reasons for better user visibility.
- [x] Add a global renderer runtime Zustand store for sync state/notices and wire sync IPC events through it.
- [x] Wire optimistic read mutation lifecycle into the global runtime store (pending/failed/clear).
- [x] Move account runtime state (`accounts`, selected account, unread counters) to a dedicated global Zustand store and refactor `useAccounts` to use it.
- [x] Move mail sync runtime UI state (`syncStatusText`, syncing account IDs) to global runtime store via `useMailSyncStatus`.
- [x] Move mailbox folder runtime state (`accountFoldersById`, selected folder path) to a global Zustand store and wire mail page setters to it.
- [x] Move mail message-list runtime state (`messages`, search results/loading, pagination flags) to a global Zustand store and wire `MailPage` setters to it.
- [x] Make single-message archive user-visible optimistic (instant local removal, background server sync, reload reconciliation on failure).
- [x] Move mail page account runtime state (`accounts`, selected account) to global `accountsRuntimeStore` with updater-compatible setters.
- [x] Move mail selection runtime state (`selectedMessageId`, selectedMessageIds, pending auto-read) to a global Zustand store via `useMailSelection`.
- [x] Remove local `MailPage` folder state and make `accountFoldersById` the single source of truth with setter-compatible selected-account folder updates.
- [x] Convert message flag toggle to true optimistic background sync with rollback-on-failure (remove full reload dependency).
- [x] Add coalesced optimistic reconciliation reloads for background move/delete sync failures to self-heal local UI after server errors.
- [x] Move mail search query runtime state to global `mailMessagesStore` so message-list/search state is fully centralized.
- [x] Route single-message archive/move failure reconciliation through the same coalesced reload queue to keep optimistic flows non-blocking and deduplicated.
- [x] Consolidate renderer locale/app-settings loading through shared IPC hooks (`useSystemLocale`, `useAppSettings`) and remove page-local duplicate subscriptions in mail/calendar/message views.
- [x] Add central `MainWindowIpcBridge` to subscribe once to account/sync IPC events and populate runtime stores instead of handling those subscriptions in `MailPage`.
- [x] Add provider-driver catalog IPC contract (`main` → `preload` → `ipcClient`) and surface driver capabilities/sync/auth metadata in add-account provider selection UI.
- [x] Make add-account provider cards fully data-driven from enabled provider-driver catalog entries (auto-show enabled providers, no hardcoded provider gates).
- [x] Move main config surface to shared modules (`appConfig`, `cloudConfig`, `authServerConfig`) and remove legacy Google/Microsoft mail OAuth default-client config constants.
- [x] Split account provider drivers into dedicated per-driver files under `src/main/mail/providers/drivers/`.
- [x] Add provider-driver auto discovery/registration so provider manager loads driver modules without hardcoded constructor wiring.
- [x] Extend provider-driver catalog metadata with per-driver `logo` and source provider card icon rendering from discovered driver metadata.
- [x] Add categorical renderer account domain handles (`useAccount`/`useAccounts`) with grouped modules (`email`, `calendar`, `contacts`) instead of page-level raw IPC calls.
- [x] Add calendar and contacts runtime stores and route account-handle refresh/mutation methods through those stores.
- [x] Migrate main mail/calendar/contacts account operations to `account.email.*`, `account.calendar.*`, and `account.contacts.*` handle APIs.
- [x] Add runtime `.env` override support for AuthServer base URL and default it to `https://llama.voracious.se`.
- [x] Add provider-driver developer guide documenting contract requirements and auto-discovery registration flow.
- [x] Remove direct Google/Microsoft OAuth refresh fallback and route token refresh exclusively through `AuthServerClient`.
- [x] Add calendar-range aware OAuth ancillary sync (Google/Microsoft) and route calendar page `sync-dav` calls to selected week/month bounds.
- [x] Add Google shared-calendar organizer metadata parsing in calendar UI with per-owner event tinting and owner labels.
- [x] Render cached calendar/contact data immediately on page open and refresh from background/global account sync without page-entry DAV sync.
- [x] Move ancillary calendar/contacts sync execution to a worker thread and switch calendar/contacts account handles to page-scoped fetches (no runtime store retention).
- [x] Add per-account sync-module toggles (`email`/`contacts`/`calendar`) and gate page visibility, worker sync execution, and mail idle/sync loops by enabled modules.
